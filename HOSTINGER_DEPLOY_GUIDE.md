# คู่มือการติดตั้งแอปพลิเคชันบน Hostinger (เวอร์ชันติดตั้งง่ายที่สุดในคลิกเดียว!)

ระบบได้รับการออกแบบให้เป็น **Zero-Configuration (ไม่ต้องตั้งค่าใด ๆ)** บน Hostinger ระบบจะทำการดาวน์โหลด ติดตั้งโมดูล และคอมไพล์หน้าเว็บไซต์ทั้งหมดให้คุณโดยอัตโนมัติทันทีที่เซิร์ฟเวอร์เริ่มทำงาน!

---

## 🚀 วิธีการติดตั้งที่ง่ายที่สุด (มีขั้นตอนเดียว!)

### ขั้นตอนที่ 1: ตั้งค่า Node.js บน Hostinger hPanel
1. นำไฟล์ทั้งหมดอัปโหลดขึ้น Hostinger (แนะนำให้เชื่อมโยงผ่าน **GitHub** หรือดาวน์โหลดแบบ **ZIP** แล้วอัปโหลดไปแตกไฟล์บน Hostinger File Manager)
2. ไปที่เมนู **Node.js** หรือ **VPS / Application manager** ในหน้าควบคุม Hostinger hPanel
3. ตั้งค่าการบูตระบบดังนี้:
   * **Node.js Version:** เลือกเวอร์ชัน **18 หรือ 20 ขึ้นไป**
   * **Application Document Root:** ชี้ไปยังโฟลเดอร์ที่คุณอัปโหลดโปรเจกต์ไว้
   * **Application Startup File:** กรอกคำว่า `index.js` (พิมพ์เป็นตัวเล็กทั้งหมด)
4. คลิกปุ่ม **Save** และคลิกปุ่ม **Start / Restart** เพื่อเริ่มทำงาน

✨ **ระบบจะเริ่มทำงานแบบอัตโนมัติ 100% ดังนี้:**
- ระบบจะตรวจจับและสั่งรัน `npm install` เพื่อดาวน์โหลดไลบรารีทั้งหมดให้คุณโดยอัตโนมัติ (หากยังไม่มี)
- ระบบจะตรวจจับและสั่งรัน `npm run build` เพื่อคอมไพล์ไฟล์หน้าบ้านและหลังบ้านลงในโฟลเดอร์ `dist/` ให้โดยอัตโนมัติ (หากยังไม่มี)
- ระบบจะรันเซิร์ฟเวอร์หลัก (Express Server) เพื่อเปิดหน้าเว็บให้เข้าใช้งานได้ทันที!

---

## 🛢️ การเชื่อมต่อฐานข้อมูล Hostinger MySQL (ง่ายดายผ่านหน้าจอเว็บ)

ระบบนี้บันทึกข้อมูลอย่างปลอดภัยพร้อมกันทั้งบน **Firebase Firestore** และ **Hostinger MySQL** ของคุณ:

1. ไปที่เมนู **Databases** -> **MySQL Databases** บน Hostinger hPanel เพื่อสร้างฐานข้อมูลใหม่
2. บันทึกข้อมูล: **Host** (ส่วนใหญ่ใช้ `localhost`), **Username**, **Password**, และ **Database Name**
3. เปิดหน้าเว็บไซต์ของคุณขึ้นมา แล้วคลิกไอคอนฟันเฟือง ⚙️ **การตั้งค่าฐานข้อมูล (Database Inspector)** ที่อยู่มุมขวาบนสุดของหน้าจอ
4. กรอกข้อมูลการเชื่อมต่อ MySQL ของคุณลงไป แล้วเปิดสวิตช์ **"สร้างฐานข้อมูลอัตโนมัติถ้าไม่มีอยู่ (Auto Create Database)"**
5. คลิกปุ่ม **"บันทึกการตั้งค่า"** จากนั้นกดปุ่ม **"ซิงค์ข้อมูลทั้งหมดไปคลาวด์และ MySQL"** ที่อยู่ด้านล่างสุด
6. เซิร์ฟเวอร์จะทำหน้าที่สร้างตารางฐานข้อมูลทั้งหมดให้คุณโดยอัตโนมัติทันที พร้อมเข้าใช้งานได้ทันที!

## 🐙 วิธีแก้ไขปัญหา "เชื่อมต่อ GitHub กับ Hostinger ไม่ได้" (Git Deployment)

หากต้องการนำโค้ดเชื่อมต่อกับ **Git Manager ของ Hostinger** แต่เชื่อมต่อไม่ได้ มักเกิดจากสาเหตุดังนี้:

### สาเหตุที่ 1: คลังข้อมูล (Repository) ของคุณเป็นแบบ "Private (ส่วนตัว)"
หาก Repo เป็นแบบส่วนตัว Hostinger จะไม่สามารถเข้าไปดึงโค้ดได้โดยตรงผ่าน HTTPS URL ปกติ

**💡 วิธีแก้ไขด้วยการใช้ SSH Deploy Key:**
1. ไปที่หน้าเมนู **Git** ใน Hostinger hPanel คุณจะพบคีย์ SSH สำหรับให้ก๊อปปี้
2. คลิก **Copy SSH Key** หรือกดสร้างใหม่แล้วก๊อปปี้คีย์ไว้
3. ไปที่หน้าโปรเจกต์ของคุณบน **GitHub** -> คลิกแท็บ **Settings** ด้านบนขวา -> มองหาเมนูด้านซ้ายเลือก **Deploy keys**
4. คลิกปุ่ม **Add deploy key** สีเขียว กรอกข้อมูล:
   * **Title:** พิมพ์ `Hostinger Server`
   * **Key:** วางคีย์ SSH ที่คัดลอกมาจาก Hostinger ลงไป
5. กดปุ่ม **Add key** เพื่อบันทึก
6. กลับมาที่หน้า **Git** บน Hostinger hPanel ให้เปลี่ยนช่อง **Repository URL** เป็นรูปแบบ **SSH Format** (ห้ามใช้ HTTPS) เช่น:
   `git@github.com:ชื่อบัญชีผู้ใช้/ชื่อโปรเจกต์.git`
7. เลือก Branch เป็น `main` หรือ `master` แล้วกด **Create** ระบบจะดาวน์โหลดและติดตั้งโค้ดได้สำเร็จทันที!

### สาเหตุที่ 2: ต้องการให้เซิร์ฟเวอร์อัปเดตโค้ดอัตโนมัติทันทีที่ Push (Auto Deployment)
1. สังเกตลิงก์ **Auto Deployment URL (Webhook URL)** ในหน้า Git ของ Hostinger และกด Copy ไว้
2. ไปที่หน้า **GitHub** -> คลิกแท็บ **Settings** -> เลือก **Webhooks** ด้านซ้ายมือ
3. คลิกปุ่ม **Add webhook**
4. วางลิงก์ที่ก๊อปปี้มาลงในช่อง **Payload URL**
5. เลือก **Content type** เป็น `application/json`
6. กดปุ่ม **Add webhook** ยืนยัน เป็นอันเสร็จสิ้น! เมื่อคุณ Push โค้ดใหม่ เซิร์ฟเวอร์จะอัปเดตและคอมไพล์ตัวเองโดยอัตโนมัติทันที!

## 💡 แนะนำเพิ่มเติม: วิธีแก้ไขหากเซิร์ฟเวอร์ของโฮสติ้งมีหน่วยความจำ (RAM) ไม่เพียงพอในการคอมไพล์
โฮสติ้งบางตัวที่มีสเปกแรมต่ำมากอาจเกิดข้อผิดพลาดในการรันคำสั่ง `npm run build` บนเซิร์ฟเวอร์ (แอปพลิเคชันจะจำลองหน้าจอแสดงข้อผิดพลาดให้อย่างชัดเจนเมื่อเข้าใช้งาน)

**วิธีแก้ไขง่าย ๆ:**
1. ให้คุณเปิดโปรแกรมเทอร์มินัลบนเครื่องคอมพิวเตอร์ของคุณเอง
2. รันคำสั่ง `npm install` และ `npm run build` ในโฟลเดอร์โปรเจกต์ที่คอมพิวเตอร์ของคุณ
3. นำโฟลเดอร์ `dist/` และโฟลเดอร์ `node_modules/` ที่สร้างได้ในคอมพิวเตอร์อัปโหลดขึ้นไปแทนที่บนเซิร์ฟเวอร์ Hostinger โดยตรงผ่าน File Manager
4. กด Restart Node.js ในระบบ หน้าเว็บจะสามารถเข้าสู่โหมดพร้อมใช้งานได้ทันทีโดยไม่ต้องคอมไพล์บนโฮสติ้งอีกต่อไป!
